Pyto Library

Pyto has some modules that allows your scripts to interact with the app.

  • console iOS | macOS
  • mainthread iOS | macOS
  • pasteboard iOS | macOS
  • sharing iOS | macOS
  • ui iOS | macOS

Platforms

Pyto supports iOS and macOS. If you want to check for platform, use the sys.platform variable. Example: 

    if sys.platform == 'ios':
        print("Running on iOS")
    elif sys.platform in ('darwin', 'macos'):
        print("Running on Mac")
    else:
        print(f"Pyto is not compatible with {sys.platform}.")
Scripts can be executed from the iOS app, from the Notification Center widget or from a process just for the script on macOS or a native iOS app built from a script. If you want to check from where the script is executed, check the __host__ global variable. 
    if __host__ is app:
        print("Running on Pyto for iOS")
    elif __host__ is widget:
        print("Running on Notification Center widget")
    else:
        if __platform__ is iOS:
            print("Running on native iOS app")
        elif __platform__ is macOS:
            print("Running on Pyto for Mac")
        else:
            print("Unknown")
__host__ is None when running on macOS because Pyto for Mac runs scripts on a subprocess. It's also None when a native app is built from the script. It's app when it's running on Pyto for iOS and widget when running on Notification Center widget.

Native iOS app

An app for iOS can be built from a script. For that, open the editor, tap the share button, and then "Generate Xcode Project". An Xcode project will be generated so you can build it from a Mac.

Third party modules

Pyto has some bundled modules:
  • chardet
  • colorama
  • dateutil
  • distlib
  • idna
  • pip - stash iOS
  • pip macOS
  • progress
  • py-filelock
  • python-certifi
  • pytz
  • python-webencodings
  • requests
  • urllib3
Including these compiled C extensions on iOS:
  • numpy
  • pandas
  • matplotlib
  • kiwisolver

This library is used internally for accessing Objective-C classes:
  • rubicon-objc

This library is used internally for code completion:
  • jedi

On macOS, system frameworks can be imported with PyObjC, so you can do anything you can do with native code!
  • PyObjC

You can install other pure Python modules from PyPi.
iOS: Press ... on main screen and then PyPi. You can search for packages and install them.
macOS: Click Window > pip on menu bar.
Installing C extensions with only work on macOS.
