Pyto Library

Pyto has some modules that allows your scripts to interact with the app.

  • console iOS | macOS
  • mainthread iOS | macOS
  • pasteboard iOS | macOS
  • sharing iOS | macOS
  • ui iOS | macOS

Platforms

Pyto supports iOS and macOS. If you want to check for platform, use the global __platform__ variable. Example: 

    if __platform__ is iOS:
        print("Running on iOS")
    elif __platform__ is macOS:
        print("Running on Mac")
    else:
        print("Unknown platform")

Third party modules

Including three compiled C extensions on iOS:
  • numpy
  • pandas
  • matplotlib
  • kiwisolver

This library is used internally for accessing Objective-C classes:
  • rubicon-objc

This library is used internally for code completion:
  • jedi

On macOS, system frameworks can be imported with PyObjC, so you can do anything you can do with native code!
  • PyObjC

You can install other pure Python modules from PyPi.
iOS: Press pip on main screen.
macOS: Click Window > pip on menu bar.
Input will be requested. Enter arguments you want to pass to pip. For example, for installing a package, type install package_name.
Installing C extensions with only work on macOS.
